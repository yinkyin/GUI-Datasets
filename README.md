# GUI Datasets

[![Companion Website](https://img.shields.io/badge/website-GUI%20Datasets%20Hub-blue)](https://yinkyin.github.io/GUI-Datasets/)
[![Deploy companion website](https://github.com/yinkyin/GUI-Datasets/actions/workflows/deploy-pages.yml/badge.svg)](https://github.com/yinkyin/GUI-Datasets/actions/workflows/deploy-pages.yml)

A curated collection and companion website for public GUI screenshot datasets. The repository is the source of truth; the website is the interactive reading and discovery layer.

- **Repository:** structured metadata, contribution rules, issue templates, automation scripts and curation docs.
- **Website:** search, filters, charts, comparison and clear explanations for each dataset.
- **Live site:** https://yinkyin.github.io/GUI-Datasets/

## Why this repo exists

GUI data is fragmented across Hugging Face, Kaggle, GitHub, Roboflow, project pages and papers. This project organizes those resources from a research user's perspective:

1. What is inside the dataset?
2. What task is it best for?
3. How do I access it?
4. What caveats should I know before using it?

The current catalog covers web, mobile, cross-platform GUI grounding, screenshot-to-code, phishing/security and China-focused GUI datasets.

## Companion website

The website lives under `site/` and is deployed to GitHub Pages:

- URL: https://yinkyin.github.io/GUI-Datasets/
- deployment workflow: `.github/workflows/deploy-pages.yml`
- build output: `site/dist/` generated by Vite and uploaded by GitHub Actions

Features:

- search across dataset names, sources, summaries, tasks and annotations;
- filter by platform, research task and access channel;
- charts for platform coverage and task distribution;
- expandable dataset cards with strengths, caveats, annotations, modalities and links;
- compare up to four datasets side by side;
- auto-classification lab for adding a new Hugging Face/Kaggle/GitHub/paper URL;
- explicit highlighting for China-focused GUI datasets.

## Add a dataset

Use the classifier as a first-pass helper:

```bash
npm run classify -- "https://huggingface.co/datasets/openbmb/CAGUI" "Chinese Android GUI grounding agent traces bilingual instructions"
```

Then manually verify the result and update `site/src/data/datasets.ts` following `docs/DATASET_SCHEMA.md`.

Before merging a new dataset, verify:

1. license and access condition;
2. exact scale and split protocol;
3. released modalities, such as screenshot, DOM, CSS, view hierarchy, bounding boxes, traces, instructions or QA pairs;
4. canonical paper/project/download links;
5. caveats that help users avoid choosing the wrong dataset.

## Recommended dataset combinations

| Use case | Recommended datasets | Why |
| --- | --- | --- |
| Screenshot-to-code | WebSight, Vision2UI, pix2code | Pair large synthetic web data with Chinese web design examples and a classic baseline. |
| Web element detection | WebUI, UI-Elements-Detection, Roboflow Web UI | Combine DOM/CSS semantics with detector-ready boxes. |
| Mobile agent training | MobileViews, AMEX, Rico, CMGUI | Mix large-scale VH data, instruction labels, classic mobile UI data and Chinese trace supervision. |
| GUI grounding evaluation | ScreenSpot, GroundUI-18k, CAGUI | Cover cross-platform and China-focused grounding scenarios. |
| Phishing/security | Phish30k, CIRCL Phishing, Grega Phishing | Combine visual phishing screenshots with URL/HTML/brand signals where available. |

## Deployment quick start

1. Push this repository to `https://github.com/yinkyin/GUI-Datasets` on branch `main`.
2. In GitHub, open **Settings → Pages**.
3. Under **Build and deployment**, set **Source** to **GitHub Actions**.
4. Push any change to `main`, or manually run **Actions → Deploy companion website to GitHub Pages → Run workflow**.
5. Open https://yinkyin.github.io/GUI-Datasets/ after the workflow succeeds.

See `docs/GITHUB_PAGES_SETUP.md` for exact commands and troubleshooting.
